## 1. 전처리 파이프라인 전체 구조

### 1.1 처리 흐름

```
원본 텍스트 (7개 데이터셋 + UnSmile)
         │
         ▼
  ┌─── Phase 1: 텍스트 정규화 (TextNormalizer) ──────────────────┐
  │  1-1. 유니코드 정규화 (NFC)                                   │
  │  1-2. HTML 엔티티 · URL · 이메일 제거                         │
  │  1-3. 초성 약어 복원 (ㅅㅂ→씨발, ㅂㅅ→병신)                    │
  │  1-4. 숫자/기호 난독화 해제 (10발→씨발, ㅅ1발→씨발)           │
  │  1-5. 별표/특수문자 마스킹 복원 (씨*→씨발, 개*끼→개새끼)      │
  │  1-6. 반복 문자 정규화 (ㅋㅋㅋㅋㅋ→ㅋㅋㅋ, ㅎㅎㅎㅎ→ㅎㅎㅎ)  │
  │  1-7. 플레이스홀더 토큰 처리 ([질병및장애]→제거)               │
  └──────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─── Phase 2: 텍스트 정제 (TextCleaner) ──────────────────────┐
  │  2-1. 공백 정규화 (연속 공백, 탭, 개행 → 단일 공백)           │
  │  2-2. 극단적 단문 필터링 (5자 미만 제거)                       │
  │  2-3. 극단적 장문 트리밍 (512자 초과 → 앞 512자)              │
  │  2-4. 빈 텍스트 제거                                          │
  │  2-5. 비한국어 텍스트 필터링 (영어/중국어만 있는 경우)         │
  └──────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─── Phase 3: 레이블 매핑 + 검증 (LabelMapper) ──────────────┐
  │  3-1. 데이터셋별 레이블 → UnSmile 9차원 변환                  │
  │  3-2. 레이블 충돌 검사 (동일 텍스트, 다른 레이블)              │
  │  3-3. 레이블 분포 검증 (극단적 불균형 감지)                    │
  │  3-4. 매핑 신뢰도 점수 부여 (0.0~1.0)                        │
  └──────────────────────────────────────────────────────────────┘
         │
         ▼
  ┌─── Phase 4: 데이터 품질 보증 (QualityAssurance) ──────────── ┐
  │  4-1. 중복 텍스트 제거 (deduplication)                         │
  │  4-2. 데이터 소스별 가중치 부여                                │
  │  4-3. 클래스별 샘플 수 균형 조정 (AEDA + Oversampling)        │
  │  4-4. 최종 통계 리포트 생성                                    │
  └──────────────────────────────────────────────────────────────┘
         │
         ▼
  전처리 완료 데이터 → data/processed/train_phase2.csv
```

### 1.2 설계 원칙

| 원칙 | 설명 |
|------|------|

## 2. Phase 1: 텍스트 정규화 (TextNormalizer)

### 2.1 유니코드 정규화

```python
import unicodedata

def normalize_unicode(text: str) -> str:
    """NFC 정규화 — 한글 자모 조합 통일"""
    return unicodedata.normalize('NFC', text)

# 예: 'ㄱ' + 'ㅏ' (분리형) → '가' (합성형)
```

### 2.2 HTML/URL 제거

```python
import re

def remove_html_url(text: str) -> str:
    """HTML 태그, URL, 이메일 주소 제거"""
    text = re.sub(r'<[^>]+>', '', text)                    # HTML 태그
    text = re.sub(r'https?://\S+', '', text)               # URL
    text = re.sub(r'www\.\S+', '', text)                   # www URL
    text = re.sub(r'[\w\.-]+@[\w\.-]+\.\w+', '', text)    # 이메일
    return text.strip()
```

### 2.3 초성 약어 복원

한국어 인터넷에서 매우 빈번한 초성 약어를 원형으로 복원합니다.

```python
CHOSUNG_MAP = {
    # 1순위: 높은 빈도 + 높은 확신도 (반드시 욕설)
    'ㅅㅂ': '씨발',   'ㅆㅂ': '씨발',   'ㅄ': '병신',
    'ㅂㅅ': '병신',   'ㅈㄹ': '지랄',   'ㅁㅊ': '미친',
    'ㄲㅈ': '꺼져',   'ㄷㅊ': '닥쳐',   'ㅈㄴ': '존나',
    'ㅅㄲ': '새끼',   'ㄱㅅㄲ': '개새끼',
    
    # 2순위: 혐오 표현 약어
    'ㅎㄴ': '한남',   'ㄱㅊㄴ': '김치녀',
    'ㅌㄷ': '틀딱',   'ㄲㄷ': '꼰대',
}

def restore_chosung(text: str) -> str:
    """초성 약어를 원형으로 복원 (긴 패턴 우선 매칭)"""
    for abbr, full in sorted(CHOSUNG_MAP.items(), key=lambda x: -len(x[0])):
        text = text.replace(abbr, full)
    return text
```

> ⚠️ **주의**: `ㅋㅋ`, `ㅎㅎ`, `ㅠㅠ` 등 감탄사성 초성 반복은 복원하지 않습니다.
> 이들은 감정 정보를 담고 있어 혐오 판단에 도움이 될 수 있습니다.

### 2.4 숫자/기호 난독화 해제

```python
OBFUSCATION_MAP = {
    # 숫자 치환형
    '10발': '씨발', '18놈': '씨발놈', '18년': '씨발년',
    '1발': '씨발',  '10놈': '씨발놈',
    
    # 숫자+한글 혼합형
    'ㅅ1발': '씨발', '시1발': '씨발',
    'ㅂ1신': '병신', '병1신': '병신',
    
    # 영어 치환형
    'tlqkf': '씨발',   # 한글 키보드→영어 변환
    'qudtls': '병신',
    'wlfka': '지랄',
}

# 정규식 패턴: 글자 사이에 특수문자 삽입된 경우
SPLIT_CHAR_PATTERNS = [
    (r'씨[\s\.\*\-_~!@#]+발', '씨발'),
    (r'병[\s\.\*\-_~!@#]+신', '병신'),
    (r'개[\s\.\*\-_~!@#]+새[\s\.\*\-_~!@#]*끼', '개새끼'),
    (r'미[\s\.\*\-_~!@#]+친', '미친'),
    (r'지[\s\.\*\-_~!@#]+랄', '지랄'),
    (r'한[\s\.\*\-_~!@#]+남', '한남'),
    (r'김[\s\.\*\-_~!@#]+치[\s\.\*\-_~!@#]*녀', '김치녀'),
    (r'틀[\s\.\*\-_~!@#]+딱', '틀딱'),
]

def deobfuscate(text: str) -> str:
    """숫자/기호 난독화 해제"""
    for pattern, replacement in OBFUSCATION_MAP.items():
        text = text.replace(pattern, replacement)
    for regex, replacement in SPLIT_CHAR_PATTERNS:
        text = re.sub(regex, replacement, text)
    return text
```

### 2.5 별표/마스킹 복원

```python
MASKING_PATTERNS = [
    # 부분 마스킹 (가운데 글자 *로 대체)
    (r'씨\*', '씨발'),
    (r'개\*끼', '개새끼'),
    (r'병\*', '병신'),
    (r'미\*놈', '미친놈'),
    (r'느\*마', '느그엄마'),
    
    # 전체 마스킹 + 힌트
    (r'시\*+발', '씨발'),
    (r'개새\*+', '개새끼'),
]

def unmask(text: str) -> str:
    """별표/특수문자 마스킹 복원"""
    for pattern, replacement in MASKING_PATTERNS:
        text = re.sub(pattern, replacement, text)
    return text
```

### 2.6 반복 문자 정규화

```python
def normalize_repeats(text: str) -> str:
    """
    의미 없는 문자 반복을 축소
    - 같은 문자 4회 이상 반복 → 3회로 축소
    - 단, 'ㅋ', 'ㅎ', 'ㅠ', 'ㅜ', 'ㄷ' 는 3회 이상 → 3회
    """
    # 같은 문자 4+ 반복 → 3회
    text = re.sub(r'(.)\1{3,}', r'\1\1\1', text)
    return text
```

### 2.7 플레이스홀더 토큰 처리

Korean Toxic Datasets 등에서 사용되는 플레이스홀더를 처리합니다.

```python
PLACEHOLDER_PATTERN = re.compile(r'\[[\w가-힣]+\]')

def remove_placeholders(text: str) -> str:
    """[질병및장애], [지역] 등 플레이스홀더 토큰 제거"""
    return PLACEHOLDER_PATTERN.sub('', text).strip()
```

### 2.8 전체 TextNormalizer 클래스

```python
class TextNormalizer:
    """텍스트 정규화 통합 파이프라인"""
    
    def normalize(self, text: str) -> str:
        """전체 정규화 파이프라인 실행"""
        text = normalize_unicode(text)
        text = remove_html_url(text)
        text = remove_placeholders(text)
        text = restore_chosung(text)
        text = deobfuscate(text)
        text = unmask(text)
        text = normalize_repeats(text)
        return text.strip()
    
    def normalize_batch(self, texts: list) -> list:
        """배치 정규화 (로깅 포함)"""
        results = []
        changed_count = 0
        for text in texts:
            normalized = self.normalize(text)
            if normalized != text:
                changed_count += 1
            results.append(normalized)
        
        logger.info(f"정규화 완료: {len(texts)}건 중 {changed_count}건 변경됨 "
                     f"({changed_count/len(texts)*100:.1f}%)")
        return results
```

## 3. Phase 2: 텍스트 정제 (TextCleaner)

### 3.1 공백 정규화

```python
def normalize_whitespace(text: str) -> str:
    """연속 공백, 탭, 개행 → 단일 공백"""
    text = re.sub(r'\s+', ' ', text)
    return text.strip()
```

### 3.2 길이 기반 필터링

```python
def filter_by_length(df: pd.DataFrame, min_len=5, max_len=512) -> pd.DataFrame:
    """극단적 단문/장문 필터링"""
    before = len(df)
    
    # 단문 제거 (의미 있는 혐오 판단 불가)
    df = df[df['text'].str.len() >= min_len]
    short_removed = before - len(df)
    
    # 장문 트리밍 (토크나이저 max_length=128에 맞춤)
    df['text'] = df['text'].str[:max_len]
    
    logger.info(f"길이 필터링: {short_removed}건 제거 (min_len={min_len})")
    return df
```

### 3.3 비한국어 텍스트 필터링

Ko-HatefulMemes 등에서 영어 텍스트가 혼재할 수 있습니다.

```python
def filter_non_korean(df: pd.DataFrame, min_korean_ratio=0.3) -> pd.DataFrame:
    """한국어 비율이 30% 미만인 텍스트 필터링"""
    KOREAN_PATTERN = re.compile(r'[가-힣ㄱ-ㅎㅏ-ㅣ]')
    
    def korean_ratio(text):
        if not text or len(text) == 0:
            return 0.0
        korean_chars = len(KOREAN_PATTERN.findall(text))
        return korean_chars / len(text)
    
    before = len(df)
    df['_kr_ratio'] = df['text'].apply(korean_ratio)
    df = df[df['_kr_ratio'] >= min_korean_ratio]
    df = df.drop(columns=['_kr_ratio'])
    
    filtered = before - len(df)
    logger.info(f"비한국어 필터링: {filtered}건 제거 (min_ratio={min_korean_ratio})")
    return df
```

### 3.4 전체 TextCleaner 클래스

```python
class TextCleaner:
    """텍스트 정제 통합 파이프라인"""
    
    def __init__(self, min_len=5, max_len=512, min_korean_ratio=0.3):
        self.min_len = min_len
        self.max_len = max_len
        self.min_korean_ratio = min_korean_ratio
    
    def clean(self, df: pd.DataFrame) -> pd.DataFrame:
        """전체 정제 파이프라인"""
        before = len(df)
        
        # 1. 공백 정규화
        df['text'] = df['text'].apply(normalize_whitespace)
        
        # 2. 빈 텍스트 제거
        df = df[df['text'].str.strip().str.len() > 0]
        
        # 3. 길이 필터링
        df = filter_by_length(df, self.min_len, self.max_len)
        
        # 4. 비한국어 필터링
        df = filter_non_korean(df, self.min_korean_ratio)
        
        after = len(df)
        logger.info(f"정제 완료: {before}건 → {after}건 ({before-after}건 제거)")
        return df
```

### 4.3 레이블 충돌 해결 전략

동일 텍스트가 여러 데이터셋에 존재할 수 있습니다.

```
레이블 충돌 해결 규칙
─────────────────────────────────────────────────
  1. UnSmile 원본 레이블은 항상 최우선 (절대 덮어쓰지 않음)
  2. 동일 텍스트에 다른 레이블이 배정된 경우:
     a. 신뢰도 점수가 높은 쪽을 채택
     b. 신뢰도 동점일 경우 → OR 연산 (Union)
        예: 소스A=[1,0,0,...] + 소스B=[0,1,0,...] → 최종=[1,1,0,...]
     c. 모순되는 경우 (같은 위치가 0과 1) → 높은 신뢰도 우선
  3. 충돌 건수가 전체의 5% 초과 시 → 경고 로그 + 수동 검증 필요
─────────────────────────────────────────────────
```

### 4.4 레이블 분포 검증

```python
def validate_label_distribution(df: pd.DataFrame, label_cols: list):
    """레이블 분포 검증 + 경고"""
    
    for col in label_cols:
        pos_ratio = df[col].mean()
        pos_count = df[col].sum()
        
        # 경고 조건
        if pos_ratio < 0.01:
            logger.warning(f"⚠️ {col}: 양성 비율 {pos_ratio:.3f} (매우 희소)")
        if pos_count < 100:
            logger.warning(f"⚠️ {col}: 양성 샘플 {pos_count}건 (100건 미만)")
        
        logger.info(f"  {col}: {pos_count:,}건 ({pos_ratio:.1%})")
    
    # 전체 Clean 비율
    clean_ratio = (df[label_cols].sum(axis=1) == 0).mean()
    logger.info(f"  Clean (모두 0): {clean_ratio:.1%}")
```

## 5. Phase 4: 데이터 품질 보증 (QualityAssurance)

### 5.1 중복 제거 (Deduplication)

```python
def deduplicate(df: pd.DataFrame) -> pd.DataFrame:
    """
    중복 텍스트 제거 전략:
    1. 완전 중복: 정확히 같은 텍스트 → 첫 번째만 유지
    2. 정규화 후 중복: 정규화 결과가 같은 경우 → 신뢰도 높은 쪽 유지
    """
    before = len(df)
    
    # 1단계: 완전 중복 제거
    df = df.drop_duplicates(subset='text', keep='first')
    
    # 2단계: 정규화 텍스트 기준 중복 제거 (선택적)
    if 'text_normalized' in df.columns:
        # 신뢰도 높은 순으로 정렬 후 중복 제거
        df = df.sort_values('mapping_confidence', ascending=False)
        df = df.drop_duplicates(subset='text_normalized', keep='first')
    
    after = len(df)
    dup_rate = (before - after) / before * 100
    logger.info(f"중복 제거: {before}건 → {after}건 ({dup_rate:.1f}% 중복)")
    
    # 중복률 5% 초과 시 경고
    assert dup_rate < 10.0, f"중복률 {dup_rate:.1f}% (10% 초과 — 데이터 소스 검증 필요)"
    
    return df
```

### 5.2 데이터 소스별 가중치

도메인 불일치를 최소화하기 위해 타겟 도메인(UnSmile)에 높은 가중치를 부여합니다.

```python
SOURCE_WEIGHTS = {
    'unsmile':          2.0,   # 타겟 도메인: 2배 가중
    'kmhas':            1.0,   # 뉴스 댓글 도메인, 매핑 신뢰도 높음
    'kold':             0.9,   # 뉴스 댓글, 매핑 약간 복잡
    'beep':             0.8,   # 연예 뉴스 댓글
    'korean_toxic':     0.7,   # 수집 데이터, 플레이스홀더 있음
    'apeach':           0.6,   # pseudo-labeling (노이즈 가능)
    'ko_hateful_memes': 0.5,   # 멀티모달 출처, 영어 혼재
    'curse_filtering':  0.7,   # 욕설 태그 데이터
}

def apply_source_weights(df: pd.DataFrame) -> pd.DataFrame:
    """
    데이터 소스별 가중치를 sample_weight 컬럼으로 추가
    → 학습 시 loss 계산에 sample_weight 반영
    """
    df['sample_weight'] = df['source'].map(SOURCE_WEIGHTS).fillna(0.5)
    return df
```

| 항목 | 처리 |
|------|------|
| 텍스트 필드 | `comment` → `text`로 컬럼명 변경 |
| 특이사항 | `raw_labels` 필드가 복잡한 중첩 리스트 구조 |
| 주의점 | `raw_labels` 파싱 시 `target_group` 추출 로직 필수 |

### 6.3 BEEP! (Korean Hate Speech)

| 항목 | 처리 |
|------|------|

| 항목 | 처리 |

### 6.6 Korean Toxic Datasets

| 항목 | 처리 |

## 10. Phase 5: 고급 데이터 증강 (AdvancedAugmentor)

### 10.1 증강 전략 전체 구조

```

═══════════════════════════════════════════════════════════

  전처리 완료 데이터
       │
       ▼
  ┌─── Layer 1: 기본 증강 (전 클래스) ──────────────────────┐

  │  10-2. Back-Translation: Ko→En→Ko (MarianMT)            │
  │  10-3. Synonym Replacement: 유의어 대체                   │
  │  10-4. Random Swap: 어절 순서 교환                        │
  └──────────────────────────────────────────────────────────┘
       │
       ▼
  ┌─── Layer 2: 고급 증강 (소수 클래스 집중) ────────────────┐
  │  10-5. Contextual Augmentation: MLM 기반 단어 대체        │
  │  10-6. Adversarial Examples: 경계 사례 생성               │
  │  10-7. Mixup (텍스트 임베딩 수준): 샘플 간 보간           │
  └──────────────────────────────────────────────────────────┘
       │
       ▼
```

### 10.2 Back-Translation (역번역 증강)

```python
from transformers import MarianMTModel, MarianTokenizer

class BackTranslator:
    """Ko→En→Ko 역번역으로 동일 의미 다른 표현 생성"""
    
    def __init__(self):
        # Ko→En
        self.ko2en_model = MarianMTModel.from_pretrained(
            'Helsinki-NLP/opus-mt-ko-en'
        )
        self.ko2en_tokenizer = MarianTokenizer.from_pretrained(
            'Helsinki-NLP/opus-mt-ko-en'
        )
        # En→Ko
        self.en2ko_model = MarianMTModel.from_pretrained(
            'Helsinki-NLP/opus-mt-en-ko'
        )
        self.en2ko_tokenizer = MarianTokenizer.from_pretrained(
            'Helsinki-NLP/opus-mt-en-ko'
        )
    
    def augment(self, text: str, n_samples: int = 2) -> list:
        """
        역번역 기반 증강
        - beam_size를 바꿔 다양한 번역 생성
        - 욕설/혐오 표현은 번역 과정에서 완화될 수 있으므로
          원문과 유사도 검사 후 유효한 것만 반환
        """
        results = []
        for beam_size in [3, 5, 8]:
            # Ko → En
            encoded = self.ko2en_tokenizer(
                text, return_tensors='pt', max_length=128, truncation=True
            )
            en_tokens = self.ko2en_model.generate(
                **encoded, num_beams=beam_size, max_length=128
            )
            en_text = self.ko2en_tokenizer.decode(
                en_tokens[0], skip_special_tokens=True
            )
            
            # En → Ko
            encoded = self.en2ko_tokenizer(
                en_text, return_tensors='pt', max_length=128, truncation=True
            )
            ko_tokens = self.en2ko_model.generate(
                **encoded, num_beams=beam_size, max_length=128
            )
            ko_text = self.en2ko_tokenizer.decode(
                ko_tokens[0], skip_special_tokens=True
            )
            
            # 원문과 다르고, 빈 문자열이 아닌 경우만 추가
            if ko_text.strip() and ko_text.strip() != text.strip():
                results.append(ko_text.strip())
        
        return results[:n_samples]
```

**Back-Translation 적용 전략:**

| 적용 대상 | 증강 배수 | 이유 |
|-----------|-----------|------|
| `기타 혐오` (569건) | 5배 → ~2,800건 | 극심한 불균형 보정 |
| `악플/욕설` (3,143건) | 2배 → ~6,200건 | 다양한 표현 확보 |
| `성소수자` (1,041건) | 3배 → ~3,100건 | 소수 클래스 |
| `종교` (386건) | 5배 → ~1,900건 | 소수 클래스 |
| 기타 클래스 | 1배 (증강 없음) | 이미 충분 |

### 10.3 Contextual Augmentation (MLM 기반)

```python
from transformers import pipeline

class ContextualAugmentor:
    """Masked Language Model 기반 문맥적 단어 대체"""
    
    def __init__(self, model_name='beomi/kcbert-base'):
        self.fill_mask = pipeline(
            'fill-mask', model=model_name, top_k=5
        )
        self.PROTECTED_WORDS = set()  # 혐오 키워드는 보호
    
    def augment(self, text: str, n_augments: int = 2) -> list:
        """
        1. 문장에서 랜덤 1~2 어절을 [MASK]로 교체
        2. MLM으로 대체어 생성
        3. 혐오 키워드는 마스킹하지 않음 (레이블 보존)
        """
        words = text.split()
        if len(words) < 3:
            return []
        
        results = []
        for _ in range(n_augments):
            # 보호 단어가 아닌 어절 중 랜덤 선택
            candidates = [
                i for i, w in enumerate(words) 
                if w not in self.PROTECTED_WORDS
            ]
            if not candidates:
                continue
            
            idx = random.choice(candidates)
            masked = words.copy()
            masked[idx] = '[MASK]'
            masked_text = ' '.join(masked)
            
            try:
                predictions = self.fill_mask(masked_text)
                if predictions:
                    new_word = predictions[0]['token_str']
                    masked[idx] = new_word
                    results.append(' '.join(masked))
            except Exception:
                continue
        
        return results
```

```
    변형: "한남충은 다 그래 [문맥 추가]"
```

### 10.5 난독화 재생성 (Reverse-Obfuscation Augmentation)

```python
class ObfuscationAugmentor:
    """정상 욕설 텍스트를 다양한 난독화 형태로 변환"""
    
    OBFUSCATION_RULES = {
        '씨발': ['ㅅㅂ', 'ㅆㅂ', 'ㅅ1발', 'c발', '씨.발', '시발', 
                 'ㅅ발', '씌발', 'ㅆ발', '10발'],
        '병신': ['ㅂㅅ', 'ㅄ', '병ㅅ', '병.신', 'ㅂ신', '뼝신'],
        '개새끼': ['ㄱㅅㄲ', '개ㅅㄲ', '개.새.끼', 'ㄱ새끼', '개색기'],
        '지랄': ['ㅈㄹ', '지.랄', 'ㅈ랄', '지ㄹ'],
        '미친': ['ㅁㅊ', '미.친', 'ㅁ친', '미ㅊ'],
    }
    
    def augment(self, text: str, n_samples: int = 3) -> list:
        """원문의 욕설을 난독화 변형으로 교체"""
        results = []
        for word, variants in self.OBFUSCATION_RULES.items():
            if word in text:
                for variant in random.sample(variants, min(n_samples, len(variants))):
                    results.append(text.replace(word, variant))
        return results[:n_samples]
```

## 11. 성능 기대 효과

### 11.1 전처리가 F1에 미치는 영향 추정

| 전처리 기법 | 대상 | 예상 F1 개선 | 근거 |
|------------|------|-------------|------|
| 난독화 해제 (초성, 숫자, 마스킹) | `악플/욕설` | +2~3%p | 기존 미탐지 욕설의 30%가 난독화 표현 |
| 플레이스홀더 제거 | Korean Toxic | +0.5%p | 노이즈 텍스트 정제 |
| 비한국어 필터링 | Ko-HatefulMemes | +0.3%p | 영어 텍스트 노이즈 제거 |
| 매핑 신뢰도 기반 가중치 | 전체 | +1~2%p | 노이즈 샘플 영향 축소 |
| 문맥 기반 과탐지 억제 | `지역`, `연령` | +0.5~1%p | FP 감소 |
| **전처리 합산** | **전체** | **+4~7%p** | - |

### 11.2 고급 증강의 F1 기여도 추정

| 증강 기법 | 적용 대상 | 예상 F1 개선 | 근거 |
|-----------|----------|-------------|------|
| Back-Translation | 소수 클래스 전체 | +1~2%p | 동일 의미 다양한 표현으로 일반화 향상 |
| Contextual Augmentation | `기타 혐오`, `종교` | +0.5~1%p | MLM 기반 자연스러운 변형 |
| Adversarial Examples | `악플/욕설`, `기타 혐오` | +0.5~1%p | 경계 사례 학습으로 FN 감소 |
| Obfuscation Augmentation | `악플/욕설` | +1~1.5%p | 난독화 패턴 학습 |
| Hard Negative Mining | Phase 1 FN 샘플 | +0.5~1%p | 오분류 패턴 집중 학습 |
| **증강 합산** | **전체** | **+3~5%p** | - |
| **전처리 + 증강 합산** | **전체** | **+7~12%p** | - |

### 11.3 전처리 없이 단순 병합 vs 정밀 전처리 비교

```
전처리 유무에 따른 예상 성능 차이

  ────────────────────────────────────

```
