## 1. 환경 설정 문제

### 1.1 CUDA 관련 오류

**문제: `RuntimeError: CUDA out of memory`**

```
RuntimeError: CUDA out of memory. Tried to allocate X MiB
```

**해결 방법:**

```python
# 1. 배치 크기 줄이기
batch_size = 16  # 32 → 16으로 감소

# 2. 캐시 정리
import torch
torch.cuda.empty_cache()

# 3. Gradient Accumulation 사용
accumulation_steps = 4
effective_batch_size = batch_size * accumulation_steps

# 4. Mixed Precision 활성화
from torch.cuda.amp import autocast, GradScaler
scaler = GradScaler()
with autocast():
    outputs = model(input_ids, attention_mask)
```

**문제: `CUDA driver version is insufficient`**

```bash
# CUDA 버전 확인
nvidia-smi

# PyTorch CUDA 버전 확인
python -c "import torch; print(torch.version.cuda)"

# 해결: 호환되는 PyTorch 버전 설치
pip install torch==2.0.0+cu118 -f https://download.pytorch.org/whl/torch_stable.html
```

### 1.2 패키지 설치 오류

**문제: `ModuleNotFoundError: No module named 'transformers'`**

```bash
# 해결: 의존성 재설치
pip install -r requirements.txt

# 개별 설치
pip install transformers>=4.30.0
```

**문제: `bitsandbytes` 설치 오류 (Apple Silicon)**

```bash

# 해결: QLoRA 비활성화하고 일반 모드 사용
```

### 1.3 Python 버전 문제

**문제: Python 버전 호환성**

## 4. 학습 관련 문제

### 4.1 손실이 감소하지 않음

**문제: 학습 손실이 정체되거나 증가**

**원인 및 해결:**

| 원인 | 해결 방법 |
|------|-----------|
| 학습률 너무 높음 | `lr=1e-5`로 감소 |
| 배치 크기 너무 작음 | `batch_size` 증가 또는 Gradient Accumulation |
| 데이터 문제 | 전처리 재확인 |
| 과적합 | Dropout 증가, Early Stopping 적용 |

```python
# 학습률 조정
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-5)  # 2e-5 → 1e-5
```

### 4.2 F1 점수가 낮음

**문제: F1-Macro가 목표에 미달**

**해결 전략:**

1. **AEDA 증강 강화**
```python
train_df_augmented = augment_minority_classes(
    train_df=train_df,
    label_columns=label_columns,
    target_size=2500,  # 1500 → 2500
    punc_ratio=0.4     # 0.3 → 0.4
)
```

2. **임계값 최적화**
```bash
python -m src.training.optimize_thresholds
```

3. **모델 앙상블**
```python
# 더 많은 모델 추가
ensemble_probs = (probs1 + probs2 + probs3) / 3
```

### 4.3 Early Stopping 너무 빠름

**문제: 충분히 학습되기 전에 조기 종료**

```python
# patience 증가
trainer.train(num_epochs=80, patience=15)  # 10 → 15
```

## 6. 메모리 최적화

### 6.1 GPU 메모리 부족

**권장 설정 (12GB VRAM 기준):**

| 설정 | 값 |
|------|-----|
| batch_size | 16 |
| max_length | 128 |
| gradient_accumulation | 2 |
| mixed_precision | True |

```python
# Mixed Precision 학습
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()
with autocast():
    outputs = model(input_ids, attention_mask)
    loss = criterion(outputs['logits'], labels)

scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
```

## 8. 자주 묻는 질문 (FAQ)

### Q1: GPU 없이 학습할 수 있나요?

**A:** 가능하지만 매우 오래 걸립니다 (수일 소요). CPU 학습 시:

```python
device = 'cpu'
# 배치 크기와 epoch 수 줄이기 권장
```

### Q2: 새로운 혐오 카테고리를 추가하려면?

**A:** 다음 파일들을 수정해야 합니다:

1. `src/data/data_loader.py`: `label_columns` 리스트에 추가
2. `src/models/model.py`: `num_labels` 수정
3. 데이터셋: 새 레이블 컬럼 추가

### Q3: 다른 한국어 모델을 사용하려면?

**A:** `models_config`에 새 모델 추가:

```python
{
    'name': 'new_model',
    'hf_name': 'huggingface/new-korean-model',
    'use_qlora': False,
    'role': '새로운 역할'
}
```

### Q4: 학습 중단 후 재개하려면?

**A:** 체크포인트에서 재개:

```python
# 체크포인트 로드
checkpoint = torch.load('models/kcelectra.pt')
model.load_state_dict(checkpoint['model_state_dict'])
optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
start_epoch = checkpoint['epoch']
```

### Q5: 앙상블 가중치를 수동으로 조정하려면?

**A:** `optimal_thresholds.json` 또는 코드에서 직접 수정:

```python
# 수동 가중치 설정
weights = [0.4, 0.35, 0.25]  # KcELECTRA, SoongsilBERT, RoBERTa
ensemble_probs = sum(w * p for w, p in zip(weights, all_probs))
```
