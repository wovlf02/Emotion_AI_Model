## 2. 프로젝트 구조 및 모듈 역할

```
src/
├── main.py                      ← 전체 파이프라인 진입점 (여기서 모든 것을 실행)
├── config.py                    ← 전역 설정 (경로, 하이퍼파라미터, Config 클래스)
├── utils.py                     ← 공통 유틸리티 (시드 고정, 로거, 체크포인트)
│
├── data/                        ← 데이터 처리 서브패키지
│   ├── data_loader.py           ← UnSmile TSV 로딩, K-Fold 분할, CSV 저장
│   ├── dataset.py               ← PyTorch Dataset (토크나이저 래핑)
│   ├── preprocessing.py         ← 텍스트 정규화(TextNormalizer), 정제(TextCleaner), 중복 제거
│   ├── external_data_merger.py  ← 7개 외부 데이터셋 병합, UnSmile 레이블로 매핑
│   ├── aeda_augmentation.py     ← AEDA 소수 클래스 증강
│   └── verify_datasets.py       ← 외부 데이터셋 접근 가능 여부 사전 검증
│
├── models/                      ← 모델 아키텍처 서브패키지
│   ├── model.py                 ← MultiLabelClassifier (AWP, Multi-Sample Dropout 내장)
│   ├── asymmetric_loss.py       ← AsymmetricLoss (양성/음성 비대칭 패널티)
│   └── ensemble.py              ← StackingMetaLearner (LightGBM + MLP + Ridge 3중 앙상블)
│
├── training/                    ← 학습 파이프라인 서브패키지
│   ├── trainer.py               ← K-Fold 학습 루프 (AWP + R-Drop + Label Smoothing 통합)
│   ├── metrics.py               ← F1-Macro, Hamming Accuracy, 리포트 출력
│   ├── optimize_thresholds.py   ← Bayesian(Optuna)/Grid Search 임계값 최적화
│   ├── curriculum_learning.py   ← CurriculumScheduler (Easy→Medium→Hard 3단계)
│   ├── hard_negative_mining.py  ← HardNegativeMiner (FN 탐지 + 오버샘플링) + SpecialistModel
│   └── self_training.py         ← SelfTrainingPipeline (3라운드 Pseudo-Labeling)
│
└── inference/                   ← 추론 파이프라인 서브패키지
    ├── inference.py             ← InferenceEngine (TTA + Temperature Scaling)
    ├── rule_system.py           ← KeywordHintGenerator + PostProcessingCorrector
    └── error_correction.py      ← ErrorCorrectionNetwork (LightGBM 잔차 보정)
```

### 모듈 간 의존 관계

```
config.py ◄─── 모든 모듈이 참조 (경로, 하이퍼파라미터)
utils.py  ◄─── 모든 모듈이 참조 (시드, 로거, 체크포인트)

data_loader.py
    └─ dataset.py
    └─ preprocessing.py
    └─ external_data_merger.py
    └─ aeda_augmentation.py

trainer.py
    ├─ model.py
    ├─ asymmetric_loss.py
    ├─ dataset.py
    └─ curriculum_learning.py (선택)

ensemble.py
    ├─ rule_system.py (키워드 힌트)
    └─ [trainer.py의 OOF 예측 필요]

inference.py
    ├─ model.py
    ├─ rule_system.py
    └─ error_correction.py (선택)

main.py ─── 위 모든 모듈을 6-Stage로 조합하여 실행
```

### 실행 흐름 (6-Stage)

```
python -m src.main
        │
        ▼
┌─────────────────────────────────────────────────────────┐
│  Stage A: 데이터 준비                                    │
│  · UnSmile 로딩 → 테스트셋 10% 선분리 (학습에 미사용)    │
│  · 7개 외부 데이터셋 병합 (최대 ~204,000건)              │
│  · 텍스트 정규화/정제/중복 제거                          │
│  · AEDA 소수 클래스 증강                                 │
│  출력: data/processed/phase2_train_merged.csv            │
│        data/processed/test_holdout.csv                   │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Stage B: 5-Model × 5-Fold K-Fold 학습                   │
│  · 5개 모델(KcELECTRA, KcBERT, KLUE-BERT,               │
│    KLUE-RoBERTa, KR-ELECTRA) × 5-Fold = 25 서브모델     │
│  · Curriculum Learning 스케줄링 (Easy→Medium→Hard)       │
│  · AWP + R-Drop + Multi-Sample Dropout + Label Smoothing │
│  출력: models/kcelectra_fold0.pt ~ kr_electra_fold4.pt   │
│        models/oof_predictions.npy                        │
│        models/oof_labels.npy                             │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Stage C: Hard Negative Mining                           │
│  · OOF 예측 기반 FN(False Negative) 샘플 탐지           │
│  · 3× 오버샘플링 DataLoader 생성                        │
│  · 취약 클래스(기타 혐오, 연령) Specialist 모델 학습     │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Stage D: Self-Training + Stacking Meta-Learner          │
│  · 3라운드 Pseudo-Labeling (비레이블 데이터 활용)        │
│    - Round 1: threshold=0.95                             │
│    - Round 2: threshold=0.92                             │
│    - Round 3: threshold=0.90                             │
│  · 품질 필터 (entropy ≤ 2.0, 동의율 ≥ 60%)             │
│  · OOF 예측으로 Stacking Meta-Learner 학습               │
│    - LightGBM (가중치 50%)                               │
│    - MLP (가중치 30%)                                    │
│    - Ridge Regression (가중치 20%)                       │
│  출력: models/meta_learner_lgbm.pkl                      │
│        models/meta_learner_mlp.pt                        │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Stage E: 룰 시스템 + Error Correction Network           │
│  · 키워드 힌트 기반 후처리 보정 (blend_weight=0.3)       │
│  · ECN: 23차원 특성 기반 LightGBM 잔차 회귀              │
│    - 입력: 앙상블 확률(9) + 키워드 힌트(9) + 텍스트 통계(5)│
│    - Safety Constraints 적용                             │
│  출력: models/ecn_class_*.pkl                            │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│  Stage F: 임계값 최적화 + 최종 평가                      │
│  · Bayesian Threshold Optimization (Optuna, 2000 trials) │
│  · 클래스별 최적 분류 임계값 산출                        │
│  · 테스트 홀드아웃셋으로 최종 성능 평가                  │
│  출력: models/thresholds.npy                             │
│        models/final_results.json                         │
└─────────────────────────────────────────────────────────┘
```

## 7. 추론 단독 실행

학습이 완료된 모델(`models/*.pt`, `models/thresholds.npy`)이 있을 때 새 텍스트에 대해 추론합니다.

### 7-1. Python 코드에서 추론

```python
from src.inference.inference import InferenceEngine
import numpy as np

# 학습된 체크포인트 경로 목록
checkpoint_paths = [
    "models/kcelectra_fold0.pt",
    "models/kcbert_fold0.pt",
    "models/klue_bert_fold0.pt",
    "models/klue_roberta_fold0.pt",
    "models/kr_electra_fold0.pt",
]

# 최적 임계값 로드
thresholds = np.load("models/thresholds.npy")

# 추론 엔진 생성
engine = InferenceEngine(
    checkpoint_paths=checkpoint_paths,
    thresholds=thresholds,
    use_tta=True,          # Test-Time Augmentation 적용
    use_rule_system=True,  # 키워드 힌트 보정 적용
)

# 단일 텍스트 추론
text = "판단하고 싶은 텍스트를 입력하세요"
result = engine.predict(text)

print("레이블별 확률:")
for label, prob in zip(result["labels"], result["probabilities"]):
    print(f"  {label}: {prob:.3f}")

print("\n최종 분류:")
for label, pred in zip(result["labels"], result["predictions"]):
    if pred:
        print(f"  ✓ {label}")
```

### 7-2. 배치 추론

```python
texts = [

### 7-3. 룰 시스템 단독 사용

from src.inference.rule_system import KeywordHintGenerator, PostProcessingCorrector
import numpy as np

# 키워드 힌트 생성

# 앙상블 확률에 키워드 힌트 반영

corrected = corrector.correct(texts, ensemble_probs)
```
