### 1.3 갭 원인 분석

전체 **13.9%p 갭**의 원인 분해:

| 원인 | 기여도 | 상세 |
|------|--------|------|
| `기타 혐오` 54% → 96% 필요 | **4.7%p** (34%) | 학습 샘플 569개, 불균형 1:25, 경계 모호 |
| `악플/욕설` 69% → 96% 필요 | **3.0%p** (21%) | 학습 샘플 3,143개이나 패턴 다양 |
| `여성/가족` 84% → 96% 필요 | **1.4%p** (10%) | 맥락 의존적 표현 미탐지 |
| `인종/국적` 85% → 96% 필요 | **1.2%p** (9%) | 변형 표현 다양 |
| 나머지 5개 클래스 | **3.6%p** (26%) | 전반적 데이터 부족 |

**핵심 결론**: 취약 2개 클래스(`기타 혐오`, `악플/욕설`)가 전체 갭의 **55%**를 차지하지만, 나머지 7개 클래스도 각각 96%까지 올려야 하므로 **데이터 규모 자체의 확대**가 필수.

## 2. Phase 2 전체 설계

### 2.1 6단계 심층 전략

```
┌─────────────────────────────────────────────────────────────────┐
│                    Phase 2: 성능 개선 전략                       │
│              (F1-Macro 82% → 98%+ 목표)                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Stage A. 외부 데이터셋 병합 + 정밀 전처리                       │
│     │  15,005건 → ~204,000건 (14배)                             │
│     │  7개 검증 완료 데이터셋 + 난독화 해제 + 품질 필터링         │
│     │                                                           │
│     ▼                                                           │
│  Stage B. 심층 앙상블 학습                                       │
│     │  5-모델 × 5-Fold = 25 서브모델                             │
│     │  AsymmetricLoss + Label Smoothing                         │
│     │  Stacking Meta-Learner (LightGBM)                        │
│     │                                                           │
│     ▼                                                           │
│  Stage C. 고급 학습 기법                                         │
│     │  Curriculum Learning (3단계 난이도 스케줄링)                │
│     │  Hard Negative Mining (FN 58% 집중 공략)                  │
│     │  AWP (Adversarial Weight Perturbation)                    │
│     │  R-Drop (정규화된 드롭아웃)                                │
│     │                                                           │
│     ▼                                                           │
│  Stage D. 반복적 자기학습 (Self-Training)                        │
│     │  모델 예측 → 고신뢰 샘플 추가 → 재학습 (3회 반복)          │
│     │  Pseudo-labeling + Confidence Filtering                   │
│     │                                                           │
│     ▼                                                           │
│  Stage E. 룰 시스템 + 에러 정정 캐스케이드                       │
│     │  키워드 힌트 주입 + 후처리 규칙                             │
│     │  Error Correction Network (경량 모델)                     │
│     │  다단계 규칙: 전처리 규칙 → 모델 → 후처리 규칙             │
│     │                                                           │
│     ▼                                                           │
│  Stage F. 추론 최적화 (Inference Optimization)                   │
│     │  Test-Time Augmentation (TTA)                             │
│     │  Temperature Scaling (Confidence Calibration)             │
│     │  Bayesian Threshold Optimization                          │
│     │                                                           │
│     ▼                                                           │
│  🎯 F1-Macro 98%+ 달성                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 예상 성능 개선 효과

| 단계 | 예상 F1-Macro | 누적 개선폭 | 핵심 개선 요소 |
|------|---------------|------------|---------------|
| 현재 (Phase 1) | 82.1% | - | 3-모델 앙상블, 15K 데이터 |
| Stage A+B 완료 | **91~93%** | +9~11%p | 데이터 14배 + Stacking 앙상블 + K-Fold |
| Stage C 완료 | **94~96%** | +2~4%p | Curriculum + Hard Negative + AWP |
| Stage D 완료 | **95~97%** | +1~2%p | Self-Training 3회 반복 |
| Stage E 완료 | **97~98%** | +1~2%p | 룰 시스템 + 에러 정정 |
| Stage F 완료 | **98~99%** | +0.5~1%p | TTA + Calibration + 임계값 최적화 |

> **설계 마진**: 목표 98%에 대해 최대 99%까지의 headroom을 확보하여,
> 일부 단계에서 예상보다 낮은 성과가 나오더라도 최종 98% 달성이 가능하도록 설계.

## 4. Stage B: 심층 앙상블 학습

### 4.1 학습 전략 변경점

| 항목 | Phase 1 | Phase 2 | 변경 이유 |
|------|---------|---------|-----------|
| **학습 데이터** | 15,005건 | ~204,000건 | 14배 확대 |
| **모델 수** | 3개 | **5개 × 5-Fold = 25 서브모델** | 다양성 극대화 |
| **앙상블 방식** | Weighted Soft Voting | **Stacking (Meta-Learner)** | 비선형 결합 |
| **Epoch** | 80 | 60 | 데이터 충분하므로 감소 |
| **Batch Size** | 32 | 32~64 | 데이터 많으므로 증가 가능 |
| **Learning Rate** | 2e-5 | 2e-5 (Cosine with Warm Restart) | 안정적 + 재시작 |
| **손실 함수** | BCEWithLogitsLoss | **AsymmetricLoss + Label Smoothing (0.05)** | 불균형 + 과적합 방지 |
| **AEDA 증강** | 2,500 목표 | 10,000 목표 | 여전히 소수 클래스 강화 |
| **추가 증강** | 없음 | **Back-Translation + EDA** | 데이터 다양성 확대 |
| **Early Stopping patience** | 10 | 12 | 충분한 수렴 대기 |
| **Gradient Accumulation** | 없음 | **2~4 steps** | 실효 배치 사이즈 128~256 |
| **Mixed Precision** | 없음 | **FP16 (AMP)** | 학습 속도 2배 |
| **예상 학습 시간** | ~15시간 | **~60~120시간** | 데이터 14배 × 5모델 × 5-Fold |

### 4.2 Stratified K-Fold 앙상블 (핵심 신규)

```
K-Fold 앙상블 구조 (5-모델 × 5-Fold)
═══════════════════════════════════════════════════════════

  데이터를 Stratified 5-Fold로 분할:
  ┌─────────┬─────────┬─────────┬─────────┬─────────┐
  │ Fold 1  │ Fold 2  │ Fold 3  │ Fold 4  │ Fold 5  │
  └─────────┴─────────┴─────────┴─────────┴─────────┘

  각 Fold에서 5개 모델 학습:
  ┌──────────────────────────────────────────────────┐
  │  Fold 1 학습 (Fold 2~5 = Train, Fold 1 = Val)   │
  │  ├─ KcELECTRA-fold1                              │
  │  ├─ KcBERT-fold1                                 │
  │  ├─ KLUE-BERT-fold1                              │
  │  ├─ KLUE-RoBERTa-fold1                           │
  │  └─ KR-ELECTRA-fold1                             │
  ├──────────────────────────────────────────────────┤
  │  Fold 2 학습 (Fold 1,3~5 = Train, Fold 2 = Val) │
  │  ├─ KcELECTRA-fold2                              │
  │  ├─ ... (동일 구조)                               │
  │  └─ KR-ELECTRA-fold2                             │
  ├──────────────────────────────────────────────────┤
  │  ... Fold 3, 4, 5 동일                           │
  └──────────────────────────────────────────────────┘

  → 총 25개 서브모델 생성
  → 추론 시 25개 모델의 예측을 Stacking으로 결합
  → K-Fold에 의해 모든 데이터가 한 번은 검증 셋으로 사용됨
```

### 4.3 Stacking Meta-Learner (핵심 신규)

기존 Soft Voting 대신 **2단계 Stacking** 앙상블을 도입합니다.

```
Stacking 앙상블 2단계 구조
═══════════════════════════════════════════════════════════

  Level 0 (Base Models):
  ──────────────────────
    25개 서브모델 (5 모델 × 5 Fold)
    각 모델이 9차원 확률 벡터 출력
    → 입력 텍스트 1개당 25 × 9 = 225차원 특성 벡터

  Level 1 (Meta-Learner):
  ──────────────────────
    입력: 225차원 특성 벡터 + 보조 특성
    보조 특성:
      - 텍스트 길이 (문자 수, 토큰 수)
      - 초성 약어 비율
      - 특수 문자 비율
      - 키워드 매칭 점수 (클래스별)
      - 모델 간 예측 일치도 (agreement score)
      - 최대 확률값 (confidence)
      - 예측 엔트로피

    모델 후보:
    ┌─────────────────────────────────────────────────┐
    │  Option A: LightGBM (권장)                       │
    │    • 빠른 학습 + 높은 성능                       │
    │    • 트리 기반 → 비선형 패턴 포착 가능            │
    │    • Feature importance로 해석 가능              │
    ├─────────────────────────────────────────────────┤
    │  Option B: 2-Layer MLP                           │
    │    • hidden=[128, 64], dropout=0.3               │
    │    • 엔드투엔드 학습 가능                        │
    │    • LightGBM보다 비선형 표현력 높음             │
    ├─────────────────────────────────────────────────┤
    │  Option C: Logistic Regression (기준선)           │
    │    • 가장 단순하고 과적합 위험 최소               │
    │    • 성능 하한선 확인용                          │
    └─────────────────────────────────────────────────┘

  Level 2 (선택적 — Blending):
  ──────────────────────
    Level 1의 출력 + Level 0의 best-single 모델 출력
    → 가중 평균으로 최종 확률 결정
    → 과적합 방지를 위한 최종 안전장치
```

**Stacking의 예상 기여**: Soft Voting 대비 **+1.5~3%p** F1 개선  
(근거: Kaggle NLP 대회에서 Stacking이 Soft Voting 대비 평균 2%p 우위)

### 4.4 학습 실행 계획

```
Phase 2 학습 순서 (구현 시)
──────────────────────────────────────────────────────────────

  5. Out-of-Fold 예측 생성 (Stacking 입력) (~30분)
  6. Meta-Learner (LightGBM) 학습 (~10분)
```

## 5. Stage C: 고급 학습 기법

Stage B에서 기본 Stacking 앙상블 학습 후, 추가 고급 학습 기법으로 성능을 끌어올립니다.

### 5.1 Curriculum Learning (단계적 학습)

```
학습 난이도 스케줄링 (3단계)
═══════════════════════════════════════════════════════════

  Phase 1: Easy Examples (Epoch 1~15)
  ├── 높은 레이블 합의도 (레이블 신뢰도 ≥ 0.95)
  ├── 명확한 혐오/정상 표현 (키워드 매칭 확실)
  └── UnSmile 원본 데이터 우선 투입

  Phase 2: Medium Examples (Epoch 16~35)
  ├── 외부 데이터셋 전체 투입
  ├── pseudo-labeled 데이터 포함
  └── 레이블 매핑 신뢰도 0.7~0.95

  Phase 3: Hard Examples (Epoch 36~60)
  ├── 맥락 의존적 혐오 표현 (욕설 없는 혐오)
  ├── 경계 사례 (혐오/비혐오 모호한 표현)
  └── 이전 epoch에서 오분류된 샘플 재주입
═══════════════════════════════════════════════════════════
```

**난이도 측정 기준**:
- 초기 모델(1~3 epoch)의 loss 값으로 샘플별 난이도 산출
- loss 상위 30% = Hard, 중간 40% = Medium, 하위 30% = Easy
- 데이터 소스별 신뢰도 가중치: UnSmile(1.0) > K-MHaS(0.9) > KOLD(0.85) > BEEP!(0.8) > 나머지(0.7)

### 5.2 Hard Negative Mining

현재 가장 큰 병목인 **False Negative (미탐지 58%)** 를 줄이기 위한 전략입니다.

```
Hard Negative Mining 파이프라인
═══════════════════════════════════════════════════════════
  Round 1:
  ─────────────────────────────────────────────────
  1. Stage B 학습 완료 모델로 전체 데이터 추론
  2. "혐오인데 Clean으로 잘못 예측한 샘플" 추출
     (True Label != 0, Predicted Label == 0)
  3. 해당 샘플을 3배 오버샘플링하여 재학습

  Round 2:
  ─────────────────────────────────────────────────
  4. Round 1 모델로 다시 추론
  5. "여전히 오분류된 샘플" 추출 (Persistent Hard Negatives)
  6. 해당 샘플에 대해 클래스별 별도 분류기 학습
     → 전문가 모델 (Specialist for 기타혐오, 악플/욕설)
  7. Specialist 모델을 Stacking에 추가

  효과:
  ─────────────────────────────────────────────────
  • `기타 혐오` FN rate: 46% → ~15% 예상
  • `악플/욕설` FN rate: 31% → ~10% 예상
  • 전체 F1 +2~3%p 기대
═══════════════════════════════════════════════════════════
```

### 5.3 AWP (Adversarial Weight Perturbation)

```
AWP 학습 전략
─────────────────────────────────────────────────
  방법:
  1. 일반 forward + backward 수행
  2. gradient 방향으로 모델 가중치에 미세 perturbation (ε=0.01)
  3. perturbation된 가중치로 다시 forward + backward
  4. 두 gradient의 평균으로 가중치 업데이트
  5. perturbation 원상 복구

  효과:
  • 입력 노이즈에 대한 강건성 향상
  • 난독화 표현 대응력 +0.5~1%p
  • Kaggle NLP 대회에서 검증된 기법
─────────────────────────────────────────────────
```

### 5.4 R-Drop (Regularized Dropout)

```
R-Drop 정규화
─────────────────────────────────────────────────
  방법:
  • 같은 입력을 두 번 forward (서로 다른 dropout mask)
  • 두 출력의 KL divergence를 추가 loss로 활용
  • L_total = L_BCE + α × L_KL (α=0.7)

  효과:
  • 정규화 효과로 일반화 성능 +0.5~1%p
  • 과적합 방지 (특히 소수 클래스)
  • 추가 계산 비용: ~30% (forward 2회)
─────────────────────────────────────────────────
```

### 5.5 Label Smoothing for Multi-Label

```
Label Smoothing 적용
─────────────────────────────────────────────────
  기존: y ∈ {0, 1}
  적용 후: y_smooth = y × (1 - ε) + ε/2
           ε = 0.05

  예시:
  • [1, 0, 0, 1, 0, ...] → [0.975, 0.025, 0.025, 0.975, 0.025, ...]

  효과:
  • 외부 데이터 매핑 노이즈에 대한 내성 향상
  • 과확신(overconfidence) 방지
  • Calibration 품질 향상 → Stage F에서 시너지
─────────────────────────────────────────────────
```

### 5.6 Multi-Task Auxiliary Loss (보조 태스크)

메인 9-class 분류 외에 보조 태스크를 추가하여 표현 학습을 강화합니다.

```
Multi-Task 구조
═══════════════════════════════════════════════════════════

  [Encoder] → [CLS hidden state]
                    │
         ┌──────────┼──────────┐
         ▼          ▼          ▼
   [Main Head]  [Binary Head]  [Severity Head]
   9-class       hate/clean    {0,1,2,3}
   multi-label   binary        4-level severity

  Main Loss:    AsymmetricLoss (9-class multi-label)
  Binary Loss:  BCE (혐오 여부 이진 분류)
  Severity Loss: CrossEntropy (혐오 강도 4단계)

  Total Loss = L_main + 0.3 × L_binary + 0.1 × L_severity
═══════════════════════════════════════════════════════════

  효과:
  • Binary head가 "혐오 여부" 판단을 보조 → FN 감소
  • Severity head가 임베딩 표현력 강화
  • F1 +0.5~1%p 기대
```

## 6. Stage D: 반복적 자기학습 (Self-Training)

Stage C까지의 모델을 활용하여 **자기학습(Self-Training)**으로 추가 성능 향상을 달성합니다.

### 6.1 Self-Training 파이프라인

```
Self-Training 반복 (3회)
═══════════════════════════════════════════════════════════

  Round 0: Stage C 완료 모델 (Teacher)
  ─────────────────────────────────────────────────
    → 기존 학습 데이터 ~204K건으로 학습 완료

  Round 1: 대규모 무레이블 데이터 Pseudo-labeling
  ─────────────────────────────────────────────────
    1. APEACH 비레이블 데이터 (11,666건) + Ko-HatefulMemes (8,500건)
       → Teacher 모델로 추론
    2. Confidence ≥ 0.95인 샘플만 선택 (약 60~70%)
    3. Pseudo-labeled 데이터를 학습 데이터에 추가
    4. 모델 재학습 → Student Model 1

  Round 2: Student 1으로 재추론
  ─────────────────────────────────────────────────
    1. Round 1에서 제외된 샘플 + Round 1의 틀린 예측 수정
    2. Confidence ≥ 0.90으로 임계값 낮춤 (더 많은 데이터)
    3. 재학습 → Student Model 2

  Round 3: 최종 정제
  ─────────────────────────────────────────────────
    1. Student 2로 전체 학습 데이터 재추론
    2. 레이블 불일치 샘플 제거 (Teacher vs Student 불일치)
    3. 정제된 데이터로 최종 학습 → Final Model

  품질 보증:
  ─────────────────────────────────────────────────
    • 매 Round 후 UnSmile test set으로 F1 측정
    • F1이 하락하면 즉시 중단 (이전 Round 모델 유지)
    • Round별 pseudo-label 오류율 샘플 검사 (200건)
═══════════════════════════════════════════════════════════
```

### 6.2 Noisy Student 변형

```
Noisy Student Training
─────────────────────────────────────────────────
  Teacher: 원본 데이터로 학습 (augmentation 없음)
  Student: 원본 + pseudo-labeled + 강한 augmentation

  Student에 적용되는 노이즈:
  • Dropout rate 0.4 (Teacher 0.3보다 높음)
  • AEDA 증강 (punc_ratio 0.4)
  • Random token masking (5%)
  • Input text에 임의 공백/특수문자 삽입

  효과:
  • Student가 Teacher보다 강건한 모델이 됨
  • 노이즈에 의해 과적합 방지
  • F1 +1~2%p 기대
─────────────────────────────────────────────────
```

## 7. Stage E: 룰 시스템 + 에러 정정 캐스케이드

### 7.1 다단계 룰 시스템

기존 단일 룰 엔진을 **3단계 다단계 룰 시스템**으로 확장합니다.

```
3단계 룰 시스템 아키텍처
═══════════════════════════════════════════════════════════

  Level 1: 전처리 룰 (Pre-Model)
  ─────────────────────────────────────────────────
    입력 → TextNormalizer (난독화 해제)
         → 정규화된 텍스트를 모델에 전달
    역할: 모델이 분석할 수 있는 깨끗한 텍스트 제공

  Level 2: 키워드 힌트 (Parallel)
  ─────────────────────────────────────────────────
    입력 → KeywordHintGenerator
         → 클래스별 키워드 매칭 점수 벡터 (9차원)
    역할: 모델 예측과 독립적인 키워드 기반 신호 제공

  Level 3: 후처리 보정 (Post-Model)
  ─────────────────────────────────────────────────
    모델 출력 + 키워드 힌트 → PostProcessingCorrector
    → 최종 확률 벡터 (블렌딩)
    → Error Correction Network (추가 보정)
    역할: 모델과 룰의 장점을 결합

═══════════════════════════════════════════════════════════
```

### 7.2 룰 사전 확장

| 룰 타입 | 기존 | 확장 | 대상 클래스 |
|---------|------|------|------------|
| 욕설 키워드 사전 | ~30개 | **200개+** | `악플/욕설` |
| 혐오 신조어 사전 | ~20개 | **100개+** | `남성`, `여성/가족`, `연령` |
| 지역 비하 사전 | ~10개 | **50개+** | `지역` |
| 종교 비하 사전 | ~5개 | **30개+** | `종교` |
| 인종 비하 사전 | ~10개 | **60개+** | `인종/국적` |
| 성소수자 비하 사전 | ~10개 | **40개+** | `성소수자` |
| **문맥 억제 규칙** | 없음 | **20개+** | 과탐지 방지 |
| **정규식 패턴** | 없음 | **50개+** | 변형 표현 캐치 |

### 7.3 Error Correction Network (ECN)

```
Error Correction Network 구조
═══════════════════════════════════════════════════════════

  입력 특성 (per sample):
  ─────────────────────────────────────────────────
    • Stacking 앙상블 출력 확률 (9차원)
    • 키워드 힌트 점수 (9차원)
    • 텍스트 통계 특성 (길이, 초성비율, 특수문자비율)
    • 모델 불확실성 (예측 엔트로피, 모델간 분산)
    → 총 ~30차원 입력

  ECN 모델:
  ─────────────────────────────────────────────────
    LightGBM Residual Corrector
    • Target: (정답 레이블 - 앙상블 예측) = residual
    • 학습 데이터: validation set의 오류 패턴
    • 최종 출력: 앙상블 확률 + ECN correction

  적용 조건:
  ─────────────────────────────────────────────────
    • ECN correction > threshold (0.1) 일 때만 적용
    • 교정 방향이 키워드 힌트와 일치할 때만 적용
    • 이중 안전장치로 과보정 방지
═══════════════════════════════════════════════════════════
```

### 7.4 적용 조건

Stage D 완료 후 성능 평가 결과에 따라 적용 강도 결정:
- F1-Macro ≥ 97%: 룰 시스템 **경량 적용** (키워드 힌트만)
- F1-Macro 95~97%: **표준 적용** (키워드 + 후처리)
- F1-Macro < 95%: **전면 적용** (키워드 + 후처리 + ECN)

## 8. Stage F: 추론 최적화 (Inference Optimization)

최종 단계로 추론 시점의 기법으로 추가 성능을 확보합니다.

### 8.1 Test-Time Augmentation (TTA)

```
TTA 전략
═══════════════════════════════════════════════════════════

  추론 시 텍스트를 여러 변형으로 복제하여 다중 추론:

  원본: "야 이 병신아 빨리 안와?"
  변형1: "야 이 병신아 빨리 안 와?"  (띄어쓰기 변형)
  변형2: "이 병신아 빨리 안와?"       (첫 단어 제거)
  변형3: "야 이 병신아 빨리 안와"     (마지막 문장부호 제거)

  3~5개 변형에 대한 예측 확률을 평균:
  P_final = mean(P_original, P_aug1, P_aug2, ...)

  효과:
  • 입력 노이즈에 대한 강건성 +0.3~0.5%p
  • 추론 시간 3~5배 증가 (배치 처리로 완화)
═══════════════════════════════════════════════════════════
```

### 8.2 Temperature Scaling (Confidence Calibration)

```
Temperature Scaling
─────────────────────────────────────────────────
  문제: 모델 출력 확률이 보정되지 않음
        (0.8 예측 ≠ 80% 확률로 실제 양성)

  방법:
  1. Validation set에서 최적 temperature T 탐색
  2. P_calibrated = σ(logits / T)
  3. T > 1이면 확률 분포가 평탄화 (과확신 완화)
  4. T < 1이면 확률 분포가 첨예화 (저확신 강화)

  클래스별 Temperature:
  • `기타 혐오`: T=1.2 (과탐지 완화)
  • `악플/욕설`: T=0.9 (미탐지 감소)
  • 나머지: T=1.0 (기본)

  효과:
  • 임계값 최적화의 효과를 극대화
  • 특히 경계 사례에서 +0.3~0.5%p
─────────────────────────────────────────────────
```

### 8.3 Bayesian Threshold Optimization

```
기존: Grid Search (0.01 간격, 9개 클래스 독립)
개선: Bayesian Optimization (Optuna 사용)
═══════════════════════════════════════════════════════════

  방법:
  1. 9개 클래스 threshold를 동시 최적화
     (Grid Search는 독립 최적화 → 클래스 간 상호작용 무시)
  2. Objective: F1-Macro 최대화
  3. Search space: [0.05, 0.95] per class
  4. Trials: 1,000~5,000회
  5. Sampler: TPE (Tree-structured Parzen Estimator)

  장점:
  • 클래스 간 threshold 상호작용 고려
  • Grid Search보다 효율적 (300배 적은 탐색)
  • 전역 최적해에 더 가까운 결과

  효과:
  • Grid Search 대비 +0.3~0.5%p
  • 특히 `기타 혐오` + `악플/욕설` 동시 최적화에서 이점
═══════════════════════════════════════════════════════════
```

### 8.4 Snapshot Ensemble (추가 앙상블 레이어)

```
Snapshot Ensemble
─────────────────────────────────────────────────
  방법:
  • Cosine Annealing with Warm Restart 스케줄러 사용
  • 각 cycle의 수렴점에서 모델 체크포인트 저장
  • 1개 모델 학습으로 3~5개 스냅샷 확보
  • 스냅샷들의 예측을 평균

  적용:
  • 5 모델 × 5 Fold × 3 Snapshot = 75개 서브모델
  • 단, 추론 시간과 trade-off → 상위 25개만 선택

  효과:
  • 추가 학습 비용 없이 앙상블 다양성 확보
  • +0.3~0.5%p 기대
─────────────────────────────────────────────────
```

## 9. 위험 요소 및 대응

| 위험 | 발생 확률 | 영향 | 대응 |
|------|----------|------|------|
| 레이블 매핑 오류로 노이즈 유입 | 중 | F1 -2~3%p | 매핑 후 랜덤 샘플링 수동 검증 (클래스당 100건+) |
| 도메인 불일치 (뉴스 vs 커뮤니티) | 중 | 일반화 저하 | 데이터셋별 가중치 조절, 원본 UnSmile ×2 가중 |
| Self-Training 노이즈 전파 | 중 | F1 하락 | 매 Round F1 체크, 하락 시 즉시 중단 |
| Stacking 과적합 (Meta-Learner) | 중 | 검증 성능 과대추정 | OOF(Out-of-Fold) 예측만 사용, 홀드아웃 검증 |
| 학습 시간 초과 (75+ 서브모델) | 중 | 일정 지연 | K-Fold 없이 단순 5-모델부터 시작 → 점진적 확장 |
| pseudo-labeling 오류 (APEACH) | 중 | 노이즈 | confidence ≥ 0.95 + Teacher/Student 일치 검증 |
| GPU 메모리 부족 | 저 | 학습 불가 | batch size 축소, gradient accumulation, FP16 |
| ECN 과보정 | 저 | FP 증가 | 이중 안전장치 (threshold + keyword 일치 조건) |

## 10. 성공 기준

| 기준 | 수치 | 판정 |
|------|------|------|
| F1-Macro | **≥ 98%** | **Phase 2 최종 목표 달성** ✅ |
| F1-Macro | 96~98% | 우수 (Stage F 튜닝으로 98% 도달 가능) |
| F1-Macro | 93~96% | 양호 (Self-Training + ECN으로 추가 개선) |
| F1-Macro | < 93% | 전략 재수립 필요 |
| `기타 혐오` F1 | ≥ 93% | 구조적 병목 해소 판정 |
| `악플/욕설` F1 | ≥ 95% | 취약 클래스 해소 판정 |
| 전 클래스 F1 | ≥ 90% | 모든 클래스 기본 품질 확보 |

### 10.1 Contingency Plan (비상 계획)

```
단계별 비상 대응

    F1 < 88% → 데이터 매핑 품질 전면 재검증
    F1 88~91% → 정상 → Stage C 진행
    F1 > 91% → 순조로움 → Stage C 간소화 가능

  ─────────────────────────────────────────────────

    F1 > 95% → Stage D 간소화 → Stage E/F로 직행

  Checkpoint 3 (Stage D 완료 후):
  ─────────────────────────────────────────────────
    F1 < 95% → 룰 시스템 전면 적용 + ECN 필수
    F1 95~97% → 정상 → Stage E 표준 적용
    F1 > 97% → Stage E 경량 적용 → Stage F로 마무리

  Checkpoint 4 (Stage E 완료 후):
  ─────────────────────────────────────────────────
    F1 < 97% → TTA + Bayesian Threshold 집중 최적화
    F1 97~98% → TTA 적용 → 98%+ 달성 가능
    F1 > 98% → 🎯 목표 달성!
═══════════════════════════════════════════════════════════
```
